#Спецификация проверки проектов с кодом

В корне обязательно есть файл README.md.
Этот файл показывает корень проекта.
Далее, должны быть корневые папки docs и specs, может быть папка tests.

Проекты бывают на Go, Rust, Typescript.
Рядом с каждым файлом кода, обязан быть файл юнит тестов этого кода.
Тесты обязаны покрывать все классы, методы и функции файла.

##Папка docs в корне
Она должна повторять структуру файлов кода, вложенную в эту папку docs, где вместо каждого файла кода одноименный html файл документации этого кода.
Код должен полностью соответствовать документации, а документация коду.

##Папка specs в корне
Она должна повторять структуру файлов кода, вложенную в эту папку specs, где вместо каждого файла одноименный md файл с спецификацией этого кода.
Код должен полностью соответствовать спецификации, а спецификация коду.

##Корневая папка tests
В ней должны быть подпапки integration, e2e.
В них должен быть полный набор тестов.
